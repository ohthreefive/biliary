## Project Overview

Retrospective service evaluation of ~500 biliary drainage and stenting procedures performed at two Glasgow hospitals (GRI and QEUH) over approximately 11 years. Analysis is in R; output is a Quarto HTML document and a deployed Shiny prediction app.

**Secondary goal:** abstract/paper submission — all outputs are publication-ready (Table 1s, forest plot, survival curves).


No `source()` calls exist within the `.R` files — each file is self-contained and they are all sourced in order by the setup chunk in `biliary.qmd`.

## Rendering

```bash
quarto render biliary.qmd --output-dir "output"
```

The setup chunk uses `cache: true` to avoid re-running the ~30 second ML tuning on every render. Clear the cache when re-running with new data:

```bash
quarto render biliary.qmd --no-cache
```

## Annual Data Update Workflow

1. Add new procedures to `data/biliary.xlsx`
2. Fix the two known data issues below if they have been corrected in the spreadsheet
3. Re-run `sources/machineLearning.R` in full to retrain the model and regenerate `shiny/model.rds`
4. Re-run `rsconnect::writeManifest()` from within `shiny/` if any packages changed
5. Commit everything (including the updated `model.rds`) and push to GitHub — Posit Connect redeploys automatically
6. Render the Quarto document: `quarto render biliary.qmd --no-cache`

## Data Dictionary (`data/biliary.xlsx`)

**Columns removed at import:** `CHI`, `DOB`, `dupCHI`, `Code`, `dateDeath`, `daysDeath`, `Column1`

`dupCHI` is validated to contain only `1` (no duplicate patients) before removal — the script halts if duplicates are detected.

**Derived columns:**
- `Year` — from `eventDate`
- `Decade` — decade bucket from `ageProcedure` (e.g. `"40s"`)
- `Site` — abbreviated hospital: `GRI` / `QEUH` (from raw codes `G107H` / `QEUH`)
- `Hospital` — full hospital name
- `Diagnosis` — `"Benign"` / `"Malignant"` (from verbose `Diag` column, which is then dropped)

**Cohort subsets** (defined in `dataWrangle.R`, available to all scripts):
- `biliary` — full dataset
- `biliary_malignant` — malignant obstruction only (`filter(Diagnosis == "Malignant")`)

**Predictor variables:** `ageProcedure`, `Sex`, `Site`/`Hospital`, `Hb` (g/L), `WCC` (×10⁹/L), `Plts` (×10⁹/L), `PT` (seconds), `Urea` (mmol/L), `Cr` (µmol/L), `Bili` (µmol/L), `Alb` (g/L), `CRP` (mg/L), `Weight` (kg — ~30% missing), `Diagnosis`, `ERCP`

**Outcome variables:** `Alive` (Yes/No), `Survival` (days — `daysSince` if alive, `daysDeath` if dead), `Inpatient mortality` (Yes/No — only meaningful when `Alive == "No"`)

All patients have ≥ 99 days follow-up (`daysSince` minimum), so no filtering is needed before calculating 30/42/90-day mortality.

## What Was Built

### descriptiveStats.R
- Shapiro-Wilk normality testing (all continuous variables non-normal → median/IQR used throughout)
- `theme_biliary` — shared ggplot2 theme (Helvetica, base_size 12, minimal); used by all scripts
- `lancet_blue <- "#0C4786"` — for single-colour charts
- Three Table 1s via `gtsummary`: by hospital (1a), sex (1b), diagnosis (1c). Mann-Whitney U / chi-squared tests. `var_labels` and `var_digits` defined once and reused.
- Procedures per year: total column chart + dodged by hospital
- `study_start` / `study_end` — formatted dates (e.g. "1st Jan 2015") from `min/max(biliary$eventDate)`; used as inline R in `.qmd`
- `avg_per_month_total` — used as inline R in `.qmd`
- Age histogram (decade buckets)

### mortality.R
- `mortality_rate()` helper function
- 30/42/90-day + overall mortality by diagnosis and total (`table_mortality`, `chart_mortality`)
- Inpatient mortality — denominator is ALL procedures (`table_inpatient`)
- Mortality by age decade, malignant cohort only (`table_decade_mortality`)

### survival.R
- 5 Kaplan-Meier analyses via `ggsurvplot` (survminer)
- `median_overall` — extracted from overall fit; used as x-axis limit for all stratified plots
- Overall survival: x-axis in years, median line shown
- Stratified analyses (all malignant cohort except diagnosis): by diagnosis, sex, hospital, ERCP
- All charts: `surv.scale = "percent"`, `risk.table = "nrisk_cumevents"`, `censor.shape = 124`, `pval = FALSE`, `legend = "none"`, `palette = "lancet"`
- Strata labels cleaned (e.g. "Diagnosis=Benign" → "Benign") via `legend.labs`

### regression.R
- Restricted to malignant cohort; 2 rows with missing CRP dropped → `biliary_cox_complete`
- `cox_labels` — subset of `var_labels` excluding Weight (Weight excluded due to ~30% missingness)
- `tbl_uvregression()` → univariable Cox; `coxph()` + `tbl_regression()` → multivariable; merged with `tbl_merge()`
- Forest plot built manually via `broom::tidy()` (ggforest has a known bug with this dataset)
- `×10⁹` uses regular `^9` in ggplot labels (Unicode superscript ⁹ may not render in some fonts)

### machineLearning.R
- Binary outcome: `died_42` (42-day mortality, Yes/No)
- All 499 patients; `Diagnosis` is a predictor
- Temporal train/test split (80/20 by `eventDate` — earliest patients train, most recent test)
- Recipe: median imputation for Weight, dummy encoding, normalisation, zero-variance removal
- Three models: logistic regression (`glm`), random forest (`ranger`), XGBoost (`xgboost`)
- Evaluation: AUC, accuracy, sensitivity, specificity on test set
- Hyperparameter tuning: 5-fold CV, 25-combination grid (mtry 2–12, min_n 2–20) → tuning showed marginal improvement; untuned RF used for deployment
- Variable importance (`vip`): `vip_labels` maps recipe-encoded names (e.g. `ERCP_Yes`) to readable labels
- Calibration plot (`probably::cal_plot_breaks`): well-calibrated to ~37.5%; x-axis clipped at 75%
- Final model trained on full dataset → saved as `list(model, baselines)` to `shiny/model.rds`

### shiny/app.R
- Loads `model.rds` (path relative to app directory — `"model.rds"`, not `"shiny/model.rds"`)
- Inputs: age, sex, diagnosis, ERCP, weight (optional), and 9 blood tests
- Output: predicted 42-day mortality %, colour-coded green/orange/red vs. diagnosis-specific baseline
- Baselines (overall, benign, malignant) pulled from saved data — update automatically on model refresh
- Deployed to Posit Connect Cloud via GitHub; pushes trigger automatic redeployment
